# Email Manager Pro Website

Static website files for Google OAuth verification and product support pages.

## Files

- `index.html`: product homepage
- `privacy.html`: public privacy policy
- `terms.html`: public terms of use

## Recommended GitHub Pages setup

Use your GitHub account:

```text
https://github.com/Ath-coding
```

Create a new public repository, for example:

```text
email-manager-pro
```

Upload the contents of this `website` folder to that repository, then enable GitHub Pages:

```text
Settings -> Pages -> Build and deployment -> Deploy from a branch -> main -> /root
```

After publishing, the expected links are:

```text
Application homepage:
https://ath-coding.github.io/email-manager-pro/

Privacy Policy:
https://ath-coding.github.io/email-manager-pro/privacy.html

Terms of Use:
https://ath-coding.github.io/email-manager-pro/terms.html
```

Use those links in Google Cloud OAuth verification.

If you instead create a repository named `Ath-coding.github.io`, the links become:

```text
https://ath-coding.github.io/
https://ath-coding.github.io/privacy.html
https://ath-coding.github.io/terms.html
```
